当任务需要图表时，默认输出高质量 ASCII，而不是只写散文说明。

## 先判断图类型

- 模块关系或分层 → 模块关系图
- 调用顺序或交互链路 → 调用链 / 时序图
- 条件分支或执行路径 → 决策流程图
- 状态迁移 → 状态机
- 配置差异或能力差异 → 对比表 / 配置矩阵

## 图表规则

1. 放在 Markdown 代码块中
2. 对齐优先，标签长度要统一处理
3. 优先控制宽度，不要为了完整性把图画到难以阅读
4. 连线保持一致：
   - 水平用 `──`
   - 垂直用 `│`
   - 分叉用 `├` `└`
   - 箭头统一用 `→` 或 `──▶`
5. 图中不要混用太多风格
6. 图后补 2 到 4 条关键解释，不重复图中文字

## 质量门槛

- 列宽必须肉眼可读
- 节点命名简洁但可辨识
- 若单图过大，拆成两个小图
- 表格列数过多时，优先拆表或改成流程图/关系图

## 输出顺序

1. 先判断最适合的图类型
2. 用一句话说明为什么选这种图
3. 再画图
4. 最后用简短说明解释关键路径与例外点

## 禁止事项

- 图和说明重复堆砌
- 为了画图牺牲可读性
- 明明更适合表格却硬画流程图
